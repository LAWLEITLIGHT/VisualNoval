# Visual Novel by白桃 — 完整使用说明书

> 适用版本：v1.10.0 起 ｜ 仓库：https://github.com/LAWLEITLIGHT/VisualNoval

这是一个 SillyTavern 第三方扩展，把"视觉小说阅读器 + 功能系统(电话/浏览器/中介所/壁纸等 app)"集成进酒馆。本说明书按"安装 → 入口 → 设置项 → 功能系统 → 常见问题"的顺序讲清每一项。

---

## 一、安装与更新

### 安装
SillyTavern → 扩展(Extensions) → **Install extension** → 填 Git URL：
```
https://github.com/LAWLEITLIGHT/VisualNoval
```
分支留空(默认 main)，选「只给我安装」或「给所有人安装」。装完 **F5 刷新**。

### 更新
扩展面板里找到 **Visual Novel** → 点 **Update** → **F5 刷新**。版本号会自动同步到启动器上的显示。

## 二、三个入口

### 1) 消息里的启动器卡片
每条含"正文 tag"(默认 `<content>…</content>`)的 AI 消息，下面会内联出现一张液态玻璃启动器卡片，上面有 4 个模式按钮：

| 按钮 | 作用 |
|---|---|
| 电脑 | PC 浮窗模式阅读器 |
| 手机 | 手机浮窗模式阅读器 |
| 网页全屏 | 占满网页的阅读器(不调用浏览器全屏) |
| 全屏 | 浏览器真全屏阅读器 |

### 2) 悬浮按钮(可拖动)
屏幕上的液态玻璃圆形按钮，**可拖动**(拖动时自动记忆位置)：

- **Visual Novel 按钮**(对话框+爱心图标)：全屏打开**最新一轮** VN。
- **功能系统 按钮**(小电脑图标)：在酒馆界面直接打开**功能系统**。

是否显示这两个按钮，在扩展设置里各自有开关。

### 3) 魔法棒菜单
输入框旁的魔法棒/扩展菜单里有 **Visual Novel** 一项，点击=全屏打开最新一轮。

> 注：Visual Novel 按钮 / 魔法棒入口 **不再要求消息里一定有正文 tag**——抓不到内容也能打开(只是内容为空)。

## 三、扩展设置项(扩展程序面板 → Visual Novel)

点开 Visual Novel 抽屉，从上到下：

### 启用 Visual Novel（总开关）
关掉后：移除所有启动器并**还原正文**、隐藏悬浮按钮、移除功能系统运行时——整个扩展不生效。重新勾上即恢复。

### 隐藏正文（只显示启动器界面）
- **勾选(默认)**：含正文 tag 的消息只显示启动器卡片，原始正文被隐藏，界面干净。
- **取消**：正文和启动器同时显示。

### 显示 Visual Novel 悬浮按钮 / 显示 功能系统 悬浮按钮
分别控制两个悬浮按钮是否出现。可以只开一个、都开、或都关。

### 全屏打开最新一轮 / 打开功能系统
两个快捷按钮，等同于点对应的悬浮按钮。

### 重置功能系统(假死时用)
万一功能系统点不动了，点它**不用刷新整页**就能重建运行时恢复。

### 识别正文用 tag
自定义"正文从 X 到 Y"。默认 `<content>` 到 `</content>`。
改成别的(比如 `<story>` 到 `</story>`)后，扩展就按新 tag 去消息里抓正文、生成启动器。

### 不显示（VN 模式里隐藏这些 tag 之间的内容）
一个可增删的列表：每条是"从 X 到 Y"，凡是落在这对 tag 之间的内容，在 VN 阅读器里**不显示**。
- 默认已内置：`<image>`→`</image>`、`<imgthink>`→`</imgthink>`。
- 点「+ 添加一条」可加你自己的(比如 `<status>`→`</status>`)，点「删」移除。

## 四、功能系统

功能系统是底部那块液态玻璃面板(代码内部名为 statusbar)，里面装着你的 app(浏览器/电话/中介所/壁纸…)。

### 特点
- **常驻单例**：开/关 VN、切全屏都**不会重建或中断**它。比如正在打电话，切 VN 不会断线。
- **里外同一个**：在 VN 阅读器里打开和用悬浮按钮在酒馆界面打开，是**同一个实例、同一份数据**(存在 `localStorage['vnm-statusbar-v2']`)。
- **默认预装 app**：浏览器、电话、中介所、壁纸 开箱即装好(每个 app 里的 API Key / 模型等设置仍需你在 app 设置页填一次)。

### 操作
- 右上角的**叉(隐藏)**：关闭功能系统面板。
- 顶部可拖动；其余按钮(刷新/设置/提示词预览等)沿用你主程序原有功能。

## 四点五、注入内容与折叠按钮（v1.11.0）

功能系统里各 app 勾选了"注入"后，它们要注入给 AI 的内容会**统一拼在你这次发送的内容前面**，一起发给大模型：

- 你在 VN 对话框输入 B 并按 Enter → 实际发出的是 `注入内容A + 你的输入B`。
- 多个 app 同时注入**不打架**，按 app 顺序依次拼接。
- 聊天里这条消息显示为：一个**液态玻璃折叠按钮「Visual Novel · 注入内容」+ 你的 B**。点开折叠按钮能看到全部注入的 A。
- 发给大模型的是 **A 紧贴 B**（去掉标签、内容完整）。

折叠靠扩展自动注册的两条酒馆正则实现(显示折叠 + 提示词剥标签)，无需手动导入。
App 作者只需在 `injectCode` 里 `return` 文本即可，细节见 `VNM-App开发指南.txt` 第七节。

## 五、阅读器内的导航

- **上一段 / 下一段**：在含启动器的各条消息(楼层)之间跳转。点"上一段"会跳到上一段的**第一句**开始。
- 四种显示模式(电脑/手机/网页全屏/全屏)和阅读器设置存在 `localStorage`，跨消息共享。
- 被 `/hide` 的楼层只要含正文 tag，也会变成启动器。

## 六、常见问题

**Q：装了 Git URL 没反应？**
A：确认装在 **扩展(Extensions)** 里、Git URL 是 `https://github.com/LAWLEITLIGHT/VisualNoval`，装完 F5。

**Q：悬浮按钮/启动器都不见？**
A：F12 控制台看有没有 `[VNM-Ext] bootstrap loaded`。没有=扩展没加载成功(重装)；有=检查"启用 Visual Novel"总开关是否被关。

**Q：功能系统点不动了？**
A：扩展设置点「重置功能系统」，或控制台 `VNM_Extension.resetSystem()`。如果重现，请把 F12 控制台里标红的 `vnm` 相关报错发给作者。

**Q：改了正文 tag 后旧消息不变？**
A：tag 改动对**新渲染**的消息生效；切换聊天或刷新可让全部按新 tag 重渲染。

## 七、控制台调试接口

```js
VNM_Extension.open()         // 全屏打开最新一轮 VN
VNM_Extension.openSystem()   // 打开功能系统
VNM_Extension.resetSystem()  // 重置功能系统
VNM_Extension.injectAll()    // 重新给所有消息注入启动器
```

## 八、工程结构(给开发者)

```
app/src/        源码: app.js(主逻辑) / launcher.html / launcher.css / apps(预装app)
app/scripts/    build.js(拼回正则) / build-extension.js(生成扩展) / build-loader.js / verify.js
extension/      manifest.json 指向的 index.js(由 _bootstrap.js + app HTML 打包)
loader/         酒馆助手脚本形态(可选, 自动更新正则)
docs/           本说明书
```
改源码只动 `app/src/`，`cd app && npm run gate` 校验后提交。
